# Business Rules - Unit 3: Admin+Kitchen Frontend

## FE-BR-01: 관리자 인증 검증
- 매장명: 필수, 1~100자
- 사용자명: 필수, 1~50자
- 비밀번호: 필수, 최소 8자
- JWT 토큰은 localStorage에 저장
- 페이지 로드 시 토큰 존재 여부 확인 → 없으면 /admin/login 리다이렉트
- 16시간 만료 시 자동 로그아웃 (API 401 응답 감지)

## FE-BR-02: 대시보드 테이블 카드 상태 표시
- 빈 테이블 (활성 세션 없음): 회색 배경
- 주문 있음 (활성 세션, 대기 주문 0): 파란색 배경
- 신규 주문 대기 (pending 주문 존재): 주황색 배경 + 대기 주문 수 뱃지
- 뱃지 숫자: 해당 테이블의 pending 상태 주문 수

## FE-BR-03: 주문 상태 변경 UI 규칙
- pending → preparing: "조리 시작" 버튼 활성화
- preparing → completed: "조리 완료" 버튼 활성화
- 역방향 전이 버튼 비활성화/숨김
- completed 상태 주문은 상태 변경 불가

## FE-BR-04: 주문 삭제 규칙
- 삭제 전 확인 팝업 필수 ("정말 삭제하시겠습니까?")
- 삭제 후 테이블 총 주문액 재계산 (SSE 이벤트로 반영)

## FE-BR-05: 테이블 이용 완료 규칙
- 미결제 주문 존재 시 경고 메시지 표시 ("미결제 주문 N건, 총 X원이 있습니다. 계속하시겠습니까?")
- 확인 팝업 후 처리
- 완료 후 해당 테이블 카드 리셋 (주문 목록 비움, 뱃지 제거)

## FE-BR-06: 메뉴 폼 검증
- 메뉴명: 필수, 1~100자
- 가격: 필수, 0 이상 정수
- 카테고리: 필수, 유효한 카테고리 선택
- 설명: 선택사항
- 이미지: 선택사항, 허용 형식 jpg/jpeg/png/gif/webp, 최대 5MB
- 검증 실패 시 해당 필드에 에러 메시지 인라인 표시

## FE-BR-07: 드래그앤드롭 순서 변경
- 같은 카테고리 내에서만 순서 변경 가능
- 드롭 완료 시 즉시 API 호출 (PUT /api/menus/reorder)
- API 실패 시 이전 순서로 롤백

## FE-BR-08: 알림음 규칙
- 기본값: 활성화(ON), 볼륨 70%
- 설정은 localStorage에 저장 (브라우저별 유지)
- 브라우저 autoplay 정책: 첫 사용자 클릭/터치 후 Audio context 활성화
- 알림 트리거: SSE newOrder 이벤트 수신 시 (대시보드, 주방 모두)

## FE-BR-09: 결제 상태 관리 규칙
- 결제 상태: unpaid(미결제), paid(결제완료), failed(결제실패)
- 상태 변경 시 드롭다운 또는 버튼으로 선택
- 변경 즉시 API 호출 (PATCH /api/payments/:orderId)
- 테이블별 총 결제/미결제 금액 실시간 계산

## FE-BR-10: 과거 주문 내역 필터링
- 날짜 범위: 시작일/종료일 (DatePicker)
- 테이블 번호: 드롭다운 선택
- 주문 상태: 체크박스 (pending/preparing/completed)
- 기본값: 오늘 날짜, 전체 테이블, 전체 상태
- 필터 변경 시 자동 재조회

## FE-BR-11: 주방 디스플레이 규칙
- completed 상태 주문은 목록에서 자동 제거
- 주문 정렬: 오래된 주문 우선 (createdAt ASC)
- 카드 색상: pending=주황, preparing=파랑
- 테이블 번호, 메뉴명, 수량 필수 표시

## FE-BR-12: 반응형 디자인 브레이크포인트
- 모바일: < 768px (1열 레이아웃)
- 태블릿: 768px ~ 1023px (2열 레이아웃)
- 데스크톱: >= 1024px (3~4열 레이아웃)
- 주방 디스플레이: 최소 768px 권장 (태블릿 이상)

## FE-BR-13: SSE 연결 관리
- 페이지 진입 시 SSE 연결 수립
- 페이지 이탈 시 SSE 연결 해제 (cleanup)
- 연결 끊김 시 자동 재연결 (3초 간격, 최대 5회)
- 재연결 실패 시 "연결 끊김" 배너 표시 + 수동 재연결 버튼
