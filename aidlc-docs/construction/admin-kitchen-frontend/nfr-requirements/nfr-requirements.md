# NFR Requirements - Unit 3: Admin+Kitchen Frontend

## NFR-01: 성능 (Performance)

| 항목 | 요구사항 |
|------|----------|
| 초기 로딩 | 3초 이내 (First Contentful Paint) |
| 페이지 전환 | 500ms 이내 |
| SSE 이벤트 반영 | 이벤트 수신 후 500ms 이내 UI 업데이트 |
| 알림음 재생 | SSE 이벤트 수신 후 1초 이내 |
| 드래그앤드롭 | 60fps 유지 (jank 없음) |
| 번들 크기 | 초기 로드 500KB 이내 (gzip) |

## NFR-02: 가용성 (Availability)

| 항목 | 요구사항 |
|------|----------|
| SSE 재연결 | 연결 끊김 시 자동 재연결 (3초 간격, 최대 5회) |
| 오프라인 대응 | 연결 끊김 배너 표시 + 수동 재연결 버튼 |
| 세션 유지 | JWT localStorage 저장, 새로고침 시 유지 |
| 에러 복구 | API 실패 시 재시도 버튼 또는 에러 메시지 표시 |

## NFR-03: 보안 (Security)

| 항목 | 요구사항 |
|------|----------|
| 인증 토큰 | JWT localStorage 저장 |
| API 요청 | 모든 요청에 Authorization 헤더 자동 첨부 |
| 401 처리 | 토큰 만료 시 자동 로그아웃 + 로그인 페이지 리다이렉트 |
| 입력 검증 | 클라이언트 측 폼 검증 (서버 검증과 이중) |

## NFR-04: 유지보수성 (Maintainability)

| 항목 | 요구사항 |
|------|----------|
| 타입 안전성 | TypeScript strict mode |
| 컴포넌트 구조 | 페이지 → 컴포넌트 → 공통 컴포넌트 계층 |
| 상태 관리 | Zustand store (도메인별 분리) |
| API 계층 | Axios 인스턴스 + 서비스 모듈 분리 |
| 코드 포맷 | ESLint + Prettier |

## NFR-05: 접근성 (Accessibility)

| 항목 | 요구사항 |
|------|----------|
| 터치 타겟 | 최소 44x44px |
| 키보드 내비게이션 | 모든 인터랙티브 요소 키보드 접근 가능 |
| 색상 대비 | WCAG 2.1 AA 기준 충족 |
| 스크린 리더 | 주요 액션에 aria-label 제공 |

## NFR-06: 반응형 (Responsiveness)

| 항목 | 요구사항 |
|------|----------|
| 모바일 | < 768px (1열) |
| 태블릿 | 768px ~ 1023px (2열) |
| 데스크톱 | >= 1024px (3~4열) |
| 주방 디스플레이 | 최소 768px 권장 |
