# NFR Requirements - Backend API (Unit 1)

## NFR-01: 성능 (Performance)

| 항목 | 요구사항 |
|------|----------|
| API 응답 시간 | 일반 CRUD: 500ms 이내, 주문 생성: 2초 이내 |
| SSE 이벤트 전달 | 이벤트 발생 후 2초 이내 클라이언트 수신 |
| 동시 접속 | 테이블 10개 + 관리자 1명 + 주방 1명 (약 12 동시 연결) |
| SSE 동시 연결 | 최대 15개 (여유 포함) |
| DB 쿼리 | 단일 쿼리 200ms 이내 |

## NFR-02: 가용성 (Availability)

| 항목 | 요구사항 |
|------|----------|
| 서버 가동 | 매장 영업 시간 중 지속 가동 |
| 재시작 복구 | 서버 재시작 시 JWT 기반 세션 자동 복구 (stateless) |
| 데이터 영속성 | MySQL 기반 영구 저장 |
| SSE 재연결 | 클라이언트 자동 재연결 (EventSource 기본 동작) |

## NFR-03: 보안 (Security)

| 항목 | 요구사항 |
|------|----------|
| 비밀번호 저장 | bcrypt 해싱 (salt rounds: 10) |
| 인증 | JWT 토큰 (16시간 만료) |
| 인가 | 역할 기반 (admin, table) - 미들웨어에서 검증 |
| 로그인 보호 | 5회 실패 시 15분 잠금 |
| 입력 검증 | 모든 API 입력 서버 측 검증 |
| 파일 업로드 | 파일 형식/크기 제한 (jpg/png/gif/webp, 5MB) |

## NFR-04: 확장성 (Scalability)

| 항목 | 요구사항 |
|------|----------|
| 규모 | 단일 매장, 소규모 (수직 확장 불필요) |
| DB 연결 | Connection pool (최대 10개) |
| 메모리 | SSE 클라이언트 인메모리 관리 (소규모로 충분) |

## NFR-05: 유지보수성 (Maintainability)

| 항목 | 요구사항 |
|------|----------|
| 코드 구조 | 3-레이어 (Controller → Service → Repository) |
| 타입 안전성 | TypeScript strict mode |
| 에러 처리 | 전역 에러 핸들러, 표준화된 에러 응답 |
| 로깅 | 요청/에러 로깅 (console 기반, 소규모 적합) |

## NFR-06: 데이터 무결성 (Data Integrity)

| 항목 | 요구사항 |
|------|----------|
| 트랜잭션 | 매장 등록, 주문 생성, 이용 완료 시 트랜잭션 사용 |
| FK 제약 | 모든 외래키 관계에 참조 무결성 적용 |
| 스냅샷 | 주문 시점 메뉴명/단가 OrderItem에 저장 (메뉴 변경 영향 차단) |
