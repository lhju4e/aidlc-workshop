# User Stories

분류: User Journey-Based | AC 형식: Checklist | 세분화: Medium

---

## Journey 1: 고객 테이블 세션

### US-01: 테이블 태블릿 자동 로그인
**As a** 고객 (김민수)
**I want to** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있기를
**So that** 대기 시간 없이 즉시 메뉴를 탐색하고 주문할 수 있다

**Acceptance Criteria:**
- [ ] 관리자가 초기 설정한 매장 식별자, 테이블 번호, 테이블 비밀번호가 로컬에 저장된다
- [ ] 저장된 정보로 자동 로그인이 수행된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 표시된다
- [ ] 자동 로그인 실패 시 초기 설정 화면이 표시된다
- [ ] 16시간 세션이 생성된다

**관련 FR**: FR-01

## Journey 2: 고객 주문 여정

### US-02: 메뉴 조회 및 탐색
**As a** 고객 (김민수)
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴 화면이 기본 화면으로 표시된다
- [ ] 카테고리별로 메뉴가 분류되어 표시된다
- [ ] 각 메뉴에 메뉴명, 가격, 설명, 이미지가 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다
- [ ] 카드 형태 레이아웃으로 표시된다
- [ ] 버튼은 터치 친화적 크기(최소 44x44px)이다

**관련 FR**: FR-02

### US-03: 장바구니 관리
**As a** 고객 (김민수)
**I want to** 주문할 메뉴를 장바구니에 담고 수량을 조절하기를
**So that** 주문 전에 원하는 메뉴와 수량을 정확히 확인하고 수정할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴를 장바구니에 추가할 수 있다
- [ ] 장바구니에서 메뉴를 삭제할 수 있다
- [ ] 수량을 증가/감소할 수 있다
- [ ] 총 금액이 실시간으로 계산되어 표시된다
- [ ] 장바구니 비우기가 가능하다
- [ ] 페이지 새로고침 시에도 장바구니가 유지된다 (로컬 저장)
- [ ] 서버 전송은 주문 확정 시에만 수행된다

**관련 FR**: FR-03

### US-04: 주문 생성
**As a** 고객 (김민수)
**I want to** 장바구니의 메뉴를 주문으로 확정하기를
**So that** 주방에 주문이 전달되어 음식을 받을 수 있다

**Acceptance Criteria:**
- [ ] 주문 내역 최종 확인 화면이 표시된다
- [ ] 주문 확정 버튼으로 주문을 생성할 수 있다
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다
- [ ] 주문 성공 시 5초 후 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
- [ ] 주문 정보에 매장 식별 정보, 테이블 식별 정보, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID가 포함된다
- [ ] 해당 테이블의 첫 주문인 경우 테이블 세션이 자동 시작된다

**관련 FR**: FR-04, FR-08-3

### US-05: 주문 내역 조회 및 실시간 상태 확인
**As a** 고객 (김민수)
**I want to** 현재 테이블의 주문 내역과 상태를 실시간으로 확인하기를
**So that** 주문이 정상 접수되었는지, 현재 어떤 상태인지 알 수 있다

**Acceptance Criteria:**
- [ ] 주문이 시간 순으로 정렬되어 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 표시된다
- [ ] SSE를 통해 주문 상태가 실시간으로 업데이트된다
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 매장 이용 완료 처리된 주문은 표시되지 않는다

**관련 FR**: FR-05

## Journey 3: 관리자 매장 운영 여정

### US-06: 매장 등록 및 관리자 인증
**As a** 관리자 (박서연)
**I want to** 매장을 등록하고 관리 시스템에 로그인하기를
**So that** 매장 주문을 관리할 수 있다

**Acceptance Criteria:**
- [ ] 매장 등록 시 관리자 계정이 자동 생성된다
- [ ] 매장 식별자 + 사용자명 + 비밀번호로 로그인할 수 있다
- [ ] 로그인 성공 시 16시간 세션이 유지된다 (JWT 토큰)
- [ ] 브라우저 새로고침 시 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 비밀번호는 bcrypt로 해싱되어 저장된다
- [ ] 로그인 시도 제한이 적용된다

**관련 FR**: FR-06

### US-07: 실시간 주문 모니터링 및 상태 관리
**As a** 관리자 (박서연)
**I want to** 들어오는 주문을 실시간으로 확인하고 상태를 변경하기를
**So that** 주문을 놓치지 않고 효율적으로 처리할 수 있다

**Acceptance Criteria:**
- [ ] SSE 기반으로 주문 목록이 실시간 업데이트된다
- [ ] 테이블별 카드 형태의 그리드 레이아웃으로 표시된다
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- [ ] 주문 카드 클릭 시 전체 메뉴 목록 상세 보기가 표시된다
- [ ] 주문 상태를 변경할 수 있다 (대기중 → 준비중 → 완료)
- [ ] 신규 주문이 시각적으로 강조된다 (색상 변경, 애니메이션)
- [ ] 신규 주문 수신 시 알림음이 재생된다
- [ ] 알림음 ON/OFF 토글이 가능하다
- [ ] 2초 이내에 주문이 표시된다
- [ ] 테이블별 필터링이 가능하다

**관련 FR**: FR-07, FR-12

### US-08: 결제 상태 관리
**As a** 관리자 (박서연)
**I want to** 주문별 결제 상태를 확인하고 수동으로 결제 완료 처리하기를
**So that** 미결제 주문을 추적하고 정산할 수 있다

**Acceptance Criteria:**
- [ ] 주문별 결제 상태(미결제/결제완료/결제실패)가 표시된다
- [ ] 관리자가 수동으로 결제 상태를 변경할 수 있다
- [ ] 테이블별 총 결제 금액 및 미결제 금액이 표시된다
- [ ] 테이블 이용 완료 시 미결제 주문이 있으면 경고가 표시된다
- [ ] 결제 상태 이력이 저장된다

**관련 FR**: FR-11

## Journey 4: 관리자 테이블/메뉴 관리 여정

### US-09: 테이블 초기 설정 및 세션 관리
**As a** 관리자 (박서연)
**I want to** 테이블 태블릿을 설정하고 테이블 세션을 관리하기를
**So that** 고객이 원활하게 주문할 수 있고, 테이블 회전을 효율적으로 관리할 수 있다

**Acceptance Criteria:**
- [ ] 테이블 번호 및 비밀번호를 설정할 수 있다
- [ ] 특정 주문을 삭제할 수 있다 (확인 팝업 → 삭제 → 총 주문액 재계산)
- [ ] 테이블 이용 완료 처리를 할 수 있다 (확인 팝업 → 주문 내역 과거 이력 이동 → 리셋)
- [ ] 이용 완료 시 해당 세션의 주문 목록 및 총 주문액이 0으로 리셋된다
- [ ] 새 고객이 이전 주문 내역 없이 시작할 수 있다
- [ ] 과거 주문 내역을 조회할 수 있다 (시간 역순, 날짜 필터링)
- [ ] 각 과거 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 성공/실패 피드백이 표시된다

**관련 FR**: FR-08

### US-10: 메뉴 관리
**As a** 관리자 (박서연)
**I want to** 메뉴를 등록, 수정, 삭제하고 노출 순서를 조정하기를
**So that** 매장 메뉴를 최신 상태로 유지하고 원하는 순서로 고객에게 보여줄 수 있다

**Acceptance Criteria:**
- [ ] 카테고리별로 메뉴를 조회할 수 있다
- [ ] 메뉴를 등록할 수 있다 (메뉴명, 가격, 설명, 카테고리, 이미지)
- [ ] 메뉴를 수정할 수 있다
- [ ] 메뉴를 삭제할 수 있다
- [ ] 드래그앤드롭으로 메뉴 노출 순서를 조정할 수 있다
- [ ] 메뉴 등록/수정 시 이미지 파일을 업로드할 수 있다
- [ ] 업로드된 이미지가 로컬 서버에 저장된다
- [ ] 필수 필드 검증이 수행된다
- [ ] 가격 범위 검증이 수행된다

**관련 FR**: FR-09, FR-10

## Journey 5: 주방 조리 여정

### US-11: 주방 주문 확인 및 조리 처리
**As a** 주방 스태프 (이준호)
**I want to** 들어온 주문을 실시간으로 확인하고 조리 상태를 업데이트하기를
**So that** 주문 순서대로 효율적으로 조리하고, 완료 시 즉시 알릴 수 있다

**Acceptance Criteria:**
- [ ] SSE 기반으로 주문 목록이 실시간으로 표시된다
- [ ] 주문이 상태별로 분류된다 (대기중/준비중)
- [ ] 조리 시작(준비중) 처리를 할 수 있다
- [ ] 조리 완료 처리를 할 수 있다
- [ ] 주문이 시간 순으로 정렬된다 (오래된 주문 우선)
- [ ] 완료된 주문이 목록에서 자동 제거된다
- [ ] 테이블 번호, 메뉴명, 수량이 표시된다

**관련 FR**: FR-13
