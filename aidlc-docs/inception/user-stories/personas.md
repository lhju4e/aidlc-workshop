# User Personas

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 고객) |
| **역할** | 매장 방문 고객 |
| **인터페이스** | 테이블 태블릿 (웹 브라우저) |
| **목표** | 빠르고 편리하게 음식을 주문하고, 주문 상태를 확인하고 싶다 |
| **동기** | 직원 호출 없이 원하는 시점에 즉시 주문, 추가 주문도 자유롭게 |
| **불편사항** | 복잡한 UI, 느린 응답, 주문 상태를 알 수 없는 상황 |
| **기술 수준** | 다양 (스마트폰 사용 가능한 일반인 수준) |

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (매장 사장님) |
| **역할** | 매장 운영자 / 관리자 |
| **인터페이스** | 관리자 웹 대시보드 (PC/태블릿 브라우저) |
| **목표** | 실시간으로 주문을 파악하고, 매장을 효율적으로 운영하고 싶다 |
| **동기** | 인건비 절감, 주문 누락 방지, 테이블 회전율 향상 |
| **불편사항** | 주문 놓침, 테이블 상태 파악 어려움, 결제 확인 번거로움 |
| **기술 수준** | 중간 (기본적인 웹 서비스 사용 가능) |

## Persona 3: 주방 스태프 (Kitchen Staff)

| 항목 | 내용 |
|------|------|
| **이름** | 이준호 (주방장) |
| **역할** | 주방 조리 담당 |
| **인터페이스** | 주방 디스플레이 (KDS, 웹 브라우저) |
| **목표** | 들어온 주문을 빠르게 확인하고, 조리 순서대로 효율적으로 처리하고 싶다 |
| **동기** | 주문서 분실 방지, 조리 순서 명확화, 조리 완료 즉시 전달 |
| **불편사항** | 주문 내용 확인 어려움, 조리 우선순위 혼란, 완료 알림 수동 전달 |
| **기술 수준** | 낮음~중간 (단순한 터치 인터페이스 선호) |
