# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

| 항목 | 내용 |
|------|------|
| **User Request** | 테이블오더 서비스 구축 (고객 주문 + 관리자 모니터링) |
| **Request Type** | New Project (Greenfield) |
| **Scope** | System-wide (Frontend + Backend + DB) |
| **Complexity** | Complex (다수 사용자 유형, 실시간 통신, 세션 관리) |
| **Requirements Depth** | Comprehensive |

---

## 기술 스택

| 영역 | 기술 |
|------|------|
| **Backend** | Node.js + Express (TypeScript) |
| **Frontend** | React (TypeScript) |
| **Database** | MySQL |
| **실시간 통신** | Server-Sent Events (SSE) |
| **배포 환경** | 로컬/온프레미스 서버 |

---

## 비즈니스 컨텍스트

- **단일 매장** 운영 (다중 매장 미지원)
- **소규모** 동시 접속 (테이블 10개 이하)
- 결제, 알림, 주방 연동, 외부 시스템 연동 제외
- Security Extension 미적용 (MVP/프로토타입 수준)

---

## Functional Requirements

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리

**설명**: 고객이 별도 로그인 없이 즉시 주문 가능하도록 자동 인증

- FR-01-1: 관리자가 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- FR-01-2: 테이블 비밀번호는 보안 인증용 (복잡한 비밀번호, 무단 사용 방지)
- FR-01-3: 1회 로그인 성공 후 로컬 저장된 정보로 자동 로그인
- FR-01-4: 자동 로그인 시 16시간 세션 생성

### FR-02: 메뉴 조회 및 탐색

**설명**: 고객이 매장 메뉴를 탐색하고 선택

- FR-02-1: 메뉴 화면이 기본 화면으로 항상 표시
- FR-02-2: 카테고리별 메뉴 분류 및 표시
- FR-02-3: 메뉴 상세 정보 표시 (메뉴명, 가격, 설명, 이미지)
- FR-02-4: 카테고리 간 빠른 이동
- FR-02-5: 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

### FR-03: 장바구니 관리

**설명**: 주문 전 선택 메뉴 임시 저장 및 수정

- FR-03-1: 메뉴 추가/삭제
- FR-03-2: 수량 조절 (증가/감소)
- FR-03-3: 총 금액 실시간 계산
- FR-03-4: 장바구니 비우기
- FR-03-5: 클라이언트 측 로컬 저장 (페이지 새로고침 시에도 유지)
- FR-03-6: 서버 전송은 주문 확정 시에만 수행

### FR-04: 주문 생성

**설명**: 장바구니 메뉴를 실제 주문으로 전환

- FR-04-1: 주문 내역 최종 확인 화면
- FR-04-2: 주문 확정 버튼
- FR-04-3: 주문 성공 시 - 주문 번호 표시, 장바구니 자동 비우기, 5초 후 메뉴 화면 자동 리다이렉트
- FR-04-4: 주문 실패 시 - 에러 메시지 표시, 장바구니 유지
- FR-04-5: 주문 정보 포함: 매장 식별 정보, 테이블 식별 정보, 주문 메뉴 목록(메뉴명, 수량, 단가), 총 주문 금액, 세션 ID

### FR-05: 주문 내역 조회

**설명**: 현재 테이블의 주문 이력 확인

- FR-05-1: 주문 시간 순 정렬
- FR-05-2: 주문별 상세 정보 (주문 번호, 시각, 메뉴/수량, 금액, 상태)
- FR-05-3: 주문 상태: 대기중/준비중/완료
- FR-05-4: SSE를 통한 주문 상태 실시간 업데이트 (고객 측 포함)
- FR-05-5: 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- FR-05-6: 매장 이용 완료 처리된 주문 제외

### FR-06: 매장 관리자 인증

**설명**: 관리자가 매장 관리 시스템에 접근

- FR-06-1: 매장 식별자 + 사용자명 + 비밀번호 입력
- FR-06-2: 매장 등록 시 관리자 계정 자동 생성 (매장 등록 프로세스)
- FR-06-3: 16시간 세션 유지 (JWT 토큰 기반)
- FR-06-4: 브라우저 새로고침 시 세션 유지
- FR-06-5: 16시간 후 자동 로그아웃
- FR-06-6: 비밀번호 bcrypt 해싱
- FR-06-7: 로그인 시도 제한

### FR-07: 실시간 주문 모니터링

**설명**: 들어오는 주문을 실시간으로 확인 및 관리

- FR-07-1: SSE 기반 주문 목록 실시간 업데이트
- FR-07-2: 그리드/대시보드 레이아웃 - 테이블별 카드 형태
- FR-07-3: 각 테이블 카드에 총 주문액 표시, 최신 주문 n개 미리보기
- FR-07-4: 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- FR-07-5: 주문 상태 변경 (대기중 → 준비중 → 완료)
- FR-07-6: 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- FR-07-7: 2초 이내 주문 표시
- FR-07-8: 테이블별 필터링 기능

### FR-08: 테이블 관리

**설명**: 테이블별 주문 상태 관리 및 세션 라이프사이클

- FR-08-1: 테이블 번호 및 비밀번호 설정 (초기 설정)
- FR-08-2: 특정 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- FR-08-3: 테이블 세션 시작 = 첫 주문 생성(확정) 시점에 자동 시작
- FR-08-4: 테이블 이용 완료 처리 (확인 팝업 → 주문 내역 과거 이력 이동 → 주문 목록/총액 리셋)
- FR-08-5: 과거 주문 내역 조회 (시간 역순, 날짜 필터링, 주문 상세 정보)
- FR-08-6: 성공/실패 피드백

### FR-09: 메뉴 관리

**설명**: 메뉴 정보 동적 관리 (MVP 포함)

- FR-09-1: 메뉴 조회 (카테고리별)
- FR-09-2: 메뉴 등록 (메뉴명, 가격, 설명, 카테고리, 이미지)
- FR-09-3: 메뉴 수정
- FR-09-4: 메뉴 삭제
- FR-09-5: 메뉴 노출 순서 조정 (드래그앤드롭 UI)
- FR-09-6: 필수 필드 검증, 가격 범위 검증

### FR-10: 메뉴 이미지 업로드

**설명**: 로컬 서버에 이미지 파일 업로드

- FR-10-1: 관리자가 메뉴 등록/수정 시 이미지 파일 업로드
- FR-10-2: 로컬 서버 파일 시스템에 저장
- FR-10-3: 업로드된 이미지 URL을 메뉴 데이터에 저장
- FR-10-4: 이미지 리사이징/최적화는 제외 (원본 그대로 저장)

### FR-11: 결제 상태 관리

**설명**: 주문에 대한 결제 상태 추적 (실제 PG 연동 없이 수동 결제 확인)

- FR-11-1: 주문별 결제 상태 관리 (미결제/결제완료/결제실패)
- FR-11-2: 관리자가 수동으로 결제 상태 변경 (결제 확인 처리)
- FR-11-3: 테이블별 총 결제 금액 및 미결제 금액 표시
- FR-11-4: 테이블 이용 완료 시 미결제 주문 경고
- FR-11-5: 결제 상태 이력 저장

### FR-12: 신규 주문 알림음

**설명**: 관리자 대시보드에서 신규 주문 수신 시 소리 알림

- FR-12-1: 신규 주문 SSE 수신 시 알림음 재생
- FR-12-2: 알림음 ON/OFF 토글
- FR-12-3: 브라우저 오디오 자동 재생 정책 대응 (사용자 인터랙션 후 활성화)

### FR-13: 주방 디스플레이 (KDS)

**설명**: 주방에서 주문 목록을 실시간으로 확인하는 별도 화면

- FR-13-1: SSE 기반 주문 목록 실시간 표시
- FR-13-2: 주문 상태별 분류 (대기중/준비중)
- FR-13-3: 주방에서 조리 시작(준비중) 및 조리 완료 처리
- FR-13-4: 주문 시간 순 정렬 (오래된 주문 우선)
- FR-13-5: 완료된 주문 자동 목록 제거
- FR-13-6: 테이블 번호, 메뉴명, 수량 표시

## Non-Functional Requirements

### NFR-01: 성능
- 주문 생성 API 응답 시간 2초 이내
- SSE를 통한 실시간 주문 표시 2초 이내
- 동시 테이블 10개 이하 지원

### NFR-02: 가용성
- 로컬/온프레미스 서버 기반 운영
- 서버 재시작 시 세션 유지 (JWT 기반)

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 형태 레이아웃
- 명확한 시각적 계층 구조

### NFR-04: 데이터 무결성
- 주문 데이터 영구 저장 (MySQL)
- 장바구니 데이터 클라이언트 로컬 저장
- 세션 ID 기반 주문 그룹화

---

## 제외 범위 (Constraints)

- 결제 처리 (카드, 현금, PG사 연동) → **변경: 결제 상태 관리만 구현 (PG 연동 없음)**
- 영수증 발행/출력, 환불, 포인트/쿠폰
- OAuth, SNS 로그인, 2FA/OTP
- 이미지 리사이징/최적화
- 푸시/SMS/이메일 알림 → **변경: 소리/진동 알림만 구현 (관리자 대시보드)**
- 주방 연동, 재고 관리 → **변경: 주방 디스플레이(KDS)만 구현 (재고 관리 제외)**
- 데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어
- 배달 플랫폼, POS, 소셜 미디어, 지도/번역 API 연동
